# US Market Shock Monitor

> **This dashboard answers: "Are we in a risky macro regime?" — not "Is today going to be red?"**
>
> It is designed for **strategic positioning** (cash levels, hedges, sector allocation) over a 2–4 week horizon, not intraday trading signals.

## What This Dashboard Is For

| Question | Answer Source | This Dashboard |
|----------|---------------|----------------|
| **"Are we in a risky macro regime?"** | Yield curve, CPI, credit spreads, unemployment, Fed balance sheet | ✅ **Perfect** — these are slow-moving, regime-defining indicators |
| **"Is today going to be red?"** | S&P 500 futures, real-time VIX, overnight news | ❌ **Wrong tool** — FRED data is always 1 day behind |

FRED data has inherent lag (see table below). It tells you the **structural risk picture** as of yesterday's close, not the intraday move. Use it to answer: *"Should I be 20% cash or 60% cash over the next month?"* not *"Should I sell at 9:45am?"*

## FRED Data Lag Reference

| Indicator | FRED Lag | What You Actually Get |
|-----------|----------|----------------------|
| **VIX** | 1 day | Yesterday's close |
| **Yield Curve** | 1 day | Yesterday's Treasury close |
| **Credit Spreads** | 1 day | Yesterday's bond market |
| **CPI** | ~2 weeks | Last month's inflation |
| **Unemployment** | ~1 month | Last month's jobs |
| **Fed Balance Sheet** | Weekly | Last Wednesday's snapshot |
| **M2 Money Supply** | Weekly | Last week's money supply |

## What Changed

This version adds **live FRED data** via GitHub Actions. The workflow runs **twice daily, every day** (including weekends) at:

- **11:30 AM UTC** = **6:30 AM ET** (pre-market data refresh)
- **9:00 PM UTC** = **4:00 PM ET** (post-market data refresh)

FRED publishes some series on weekends (e.g., M2 money supply, weekly balance sheet), so weekend runs ensure you always have the freshest data available.

### Fetched Series

- **VIX** (volatility)
- **Yield Curve** (10Y-2Y spread)
- **CPI** (inflation, YoY computed)
- **Unemployment**
- **Credit Spreads** (high yield)
- **Fed Balance Sheet**
- **Consumer Sentiment**
- **US Dollar Index**
- **WTI Crude Oil**
- **Treasury Yields** (10Y & 2Y)
- **M2 Money Supply**
- **S&P 500 & NASDAQ**

Indicators without FRED coverage (Fear & Greed, National Debt, Govt Spending, Margin Debt, Geopolitical Risk) remain as demo/fallback data.

## Setup Instructions

### 1. Get a FRED API Key (free)

1. Go to https://fred.stlouisfed.org/docs/api/api_key.html
2. Request a key (instant, no approval needed)
3. Copy the key

### 2. Add the Key to GitHub Secrets

1. In your repo → **Settings** → **Secrets and variables** → **Actions**
2. Click **New repository secret**
3. Name: `FRED_API_KEY`
4. Value: your FRED API key
5. Click **Add secret**

### 3. Commit These Files

```bash
# Add the workflow
git add .github/workflows/fetch-fred.yml

# Add the updated index.html
git add index.html

# Commit
git commit -m "Add FRED live data pipeline"
git push
```

### 4. Trigger the First Run

1. Go to **Actions** tab in your repo
2. Click **Fetch FRED Data**
3. Click **Run workflow** → **Run workflow**
4. Wait ~1 minute for the workflow to complete
5. Refresh your repo — you should see a `data/` folder with JSON files
6. Your site will now display live data instead of demo data

### 5. How It Works

```
GitHub Actions (daily, 2x/day) → FRED API → data/*.json → GitHub Pages
                                          ↑
                                    your API key (secure)
```

- The API key never leaves GitHub's secure environment
- The browser loads local JSON files — no CORS issues, no exposed keys
- If FRED data is missing, the site falls back to demo data automatically
- Weekend runs catch weekly series (M2, Fed balance sheet) that update on Saturdays

## Troubleshooting

**Site still shows "demo mode"**
- The workflow hasn't run yet. Trigger it manually: Actions → Fetch FRED Data → Run workflow
- Or wait for the next scheduled run (6:30 AM ET or 4:00 PM ET)

**Some indicators are blank**
- FRED occasionally has delayed data for certain series
- The site automatically falls back to demo data for any missing series

**Workflow fails**
- Check that `FRED_API_KEY` is set correctly in repo Settings → Secrets
- Check the Actions log for specific error messages
