# US Market Shock Monitor

A real-time market shock dashboard powered by FRED (Federal Reserve Economic Data).

## What Changed

This version adds **live FRED data** via GitHub Actions. The workflow runs daily at 4pm UTC (≈ US market close) and fetches:

- **VIX** (volatility)
- **Yield Curve** (10Y-2Y spread)
- **CPI** (inflation, YoY computed)
- **Unemployment**
- **Credit Spreads** (high yield)
- **Fed Balance Sheet**
- **Consumer Sentiment**
- **US Dollar Index**
- **WTI Crude Oil**
- **Treasury Yields** (10Y & 2Y)

Indicators without FRED coverage (Fear & Greed, National Debt, Govt Spending, Margin Debt, Geopolitical Risk) remain as demo/fallback data.

## Setup Instructions

### 1. Get a FRED API Key (free)

1. Go to https://fred.stlouisfed.org/docs/api/api_key.html
2. Request a key (instant, no approval needed)
3. Copy the key

### 2. Add the Key to GitHub Secrets

1. In your repo → **Settings** → **Secrets and variables** → **Actions**
2. Click **New repository secret**
3. Name: `FRED_API_KEY`
4. Value: your FRED API key
5. Click **Add secret**

### 3. Commit These Files

```bash
# Add the workflow
git add .github/workflows/fetch-fred.yml

# Add the updated index.html
git add index.html

# Commit
git commit -m "Add FRED live data pipeline"
git push
```

### 4. Trigger the First Run

1. Go to **Actions** → **Fetch FRED Data** → **Run workflow**
2. Wait ~1 minute for the workflow to complete
3. Refresh your repo — you should see a `data/` folder with JSON files
4. Your site will now display live data instead of demo data

### 5. How It Works

```
GitHub Actions (daily) → FRED API → data/*.json → GitHub Pages
                                    ↑
                              your API key (secure)
```

- The API key never leaves GitHub's secure environment
- The browser loads local JSON files — no CORS issues, no exposed keys
- If FRED data is missing, the site falls back to demo data automatically

## Troubleshooting

**Site still shows "demo mode"**
- The workflow hasn't run yet. Trigger it manually: Actions → Fetch FRED Data → Run workflow
- Or wait for the next scheduled run (daily at 4pm UTC)

**Some indicators are blank**
- FRED occasionally has delayed data for certain series
- The site automatically falls back to demo data for any missing series

**Workflow fails**
- Check that `FRED_API_KEY` is set correctly in repo Settings → Secrets
- Check the Actions log for specific error messages

## Data Sources

All live data comes from [FRED](https://fred.stlouisfed.org/) (Federal Reserve Bank of St. Louis), which is free for public use.
